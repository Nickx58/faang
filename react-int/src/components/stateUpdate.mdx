3️⃣ React State Example (Most Important)

In React you should never mutate state directly.

const [users, setUsers] = useState([
{ id: 1, name: "John" },
{ id: 2, name: "Mike" }
]);

function addUser() {
const newUser = { id: 3, name: "Sara" };

setUsers(prev => [...prev, newUser]);
}

Key point:

setUsers(prev => [...prev, newUser])

Senior React Interview Tip 💡

Interviewers often ask:

Why is this wrong in React?

users.push(newUser);
setUsers(users);

Answer:

It mutates state

React may not detect the change

Can cause unexpected rendering bugs

Correct approach:

setUsers(prev => [...prev, newUser]);

React State Example
const [todos, setTodos] = useState([
  { id: 1, task: "Learn React", completed: false },
  { id: 2, task: "Practice DSA", completed: true }
]);

function toggleTodo(id) {
  setTodos(prev =>
    prev.map(todo =>
      todo.id === id
        ? { ...todo, completed: !todo.completed }
        : todo
    )
  );
}
